**GuideLine.swift -- capture-space storage with window-space rendering:**

1. Rename the `position` property to `capturePosition` (CGFloat). This stores the guide's position in capture-space (original unzoomed coordinates). The `init` parameter stays named `position` but assigns to `capturePosition`.

2. Add a `renderPosition(zoomState:screenSize:)` method that converts `capturePosition` to window-space for rendering. For a vertical line at capturePosition X: `windowX = (capturePosition + zoomState.panOffset.x) * zoomState.level.rawValue`. For horizontal line at capturePosition Y: `windowY = (capturePosition + zoomState.panOffset.y) * zoomState.level.rawValue`. At 1x with no pan, this is identity.

3. Update the `update()` method signature to accept `capturePosition` (the capture-space position) and add a `zoomState` parameter plus `screenSize`. Inside, compute `renderPos = renderPosition(zoomState:screenSize:)` and use `renderPos` for the line path and pill layout. Store `capturePosition` in `self.capturePosition`. The pill coordinate label (X/Y value) should display `Int(round(capturePosition))` (screen-space value), NOT the zoomed render position.

4. Update `layoutPill` to accept the `renderPos` (window-space) for pill positioning, but the `capturePosition` for the displayed value. The pill itself is positioned relative to `renderPos` (where the line appears on screen).

5. Update `shrinkToPoint` to use a `renderPosition` parameter (the caller converts the click point).

6. Update `setHovered` -- no coordinate changes needed, just visual state.

**GuideLineManager.swift -- zoom-aware operations:**

1. Add a `package var zoomState = ZoomState()` property.

2. Update `updatePreview(at:)` to accept the point already in capture-space (the caller converts). Store `currentPosition` as capture-space. Pass `zoomState` to `previewLine.update()`.

3. Update `updateHover(at:)` -- the incoming point is capture-space. In `findNearestLine(to:within:)`, compare the capture-space cursor coordinate against each line's `capturePosition`. The 5px threshold must be in screen-space, so divide the capture-space distance by the zoom scale: `let threshold = 5.0 / zoomState.level.rawValue` (at 2x zoom, 5px on screen = 2.5 points in capture-space).

4. Update `placeGuide()` to pass `zoomState` to the new GuideLine's `update()` call.

5. Update `toggleDirection()` to pass `zoomState`.

6. Update `cycleStyle(cursorPosition:)` -- the incoming cursor is capture-space; pass `zoomState` when updating preview. The `colorIndicator?.updatePosition` call needs the window-space point, so the caller should pass both capture and window points, or the manager converts internally using `capturePointToWindowPoint`. Add a second parameter `windowPoint` to `cycleStyle` for the color indicator positioning.

7. Add `package func updateForZoom(_ newZoomState: ZoomState)` that:
   - Stores the new zoom state
   - Calls `previewLine.update(...)` with current position and new zoom state (repositions preview)
   - Iterates all `placedLines` and calls a new `GuideLine.updateRenderPosition(zoomState:screenSize:)` method that recomputes only the line path from the stored `capturePosition` (no pill changes needed for placed lines since they have hidden pills)

8. Add `GuideLine.updateRenderPosition(zoomState:screenSize:)` -- recomputes the line path using `renderPosition(zoomState:screenSize:)`. Only updates the line path, not the pill.

9. Update `colorIndicator?.updatePosition(at:screenSize:)` calls -- the color indicator follows the cursor in window-space. Either pass the window-space point or convert inside the manager.

**Key principle:** All stored positions are capture-space. All rendering uses window-space derived from capture-space + zoomState. The pill displays capture-space values (true screen coordinates). Hit testing compares in capture-space with zoom-adjusted threshold.

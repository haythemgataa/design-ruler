**handleMouseMoved(to:)** — The primary fix. Convert `windowPoint` to capture-space before building the AppKit screen point for EdgeDetector:
```swift
let capturePoint = windowPointToCapturePoint(windowPoint, zoomState: zoomState, screenSize: screenBounds.size)
let appKitScreenPoint = NSPoint(
    x: screenBounds.origin.x + capturePoint.x,
    y: screenBounds.origin.y + capturePoint.y

```
Keep passing `windowPoint` (not capturePoint) to `crosshairView.update(cursor:edges:zoomScale:)` — the crosshair renders in screen-space (cursor position) with edges scaled by `zoomScale` (already implemented in CrosshairView).

**mouseDown(with:)** — Convert `windowPoint` to capture-space for:
1. Selection hit-test (`selectionManager.hitTest(capturePoint)`)
2. Edge detection after removal (convert to AppKit screen point same as handleMouseMoved)
3. Drag start (`selectionManager.startDrag(at: capturePoint)`)

**mouseDragged(with:)** — Convert `windowPoint` to capture-space for `selectionManager.updateDrag(to: capturePoint)`.

**mouseUp(with:)** — Convert `windowPoint` to capture-space for:
1. `selectionManager.endDrag(at: capturePoint, screenBounds: screenBounds)`
2. Edge detection for crosshair restoration (convert to AppKit screen point same as handleMouseMoved)

**activate(firstMoveAlreadyReceived:)** — Convert `wp` to capture-space before building the AppKit screen point:
```swift
let cp = windowPointToCapturePoint(wp, zoomState: zoomState, screenSize: screenBounds.size)
let sp = NSPoint(x: screenBounds.origin.x + cp.x, y: screenBounds.origin.y + cp.y)
```

**CRITICAL:** At 1x zoom, `windowPointToCapturePoint` is identity (returns the same point), so there is zero behavioral change at 1x. All dimension pill values remain in capture-space (already correct — `edges.left?.distance` etc. are capture-space distances). The `zoomScale` parameter on `crosshairView.update()` already handles visual scaling of edge line positions.

Add a helper method to MeasureWindow to reduce repetition:

```swift
}
```
Use this in handleMouseMoved, mouseDown (removal restore), mouseUp (restore), and activate. Drag operations use the raw capture point directly (not screen point).

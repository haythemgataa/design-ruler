  <action>
Create a new file `ZoomState.swift` in the Utilities directory containing:

**ZoomLevel enum** (package access):
- Cases: `.one` (rawValue 1.0), `.two` (rawValue 2.0), `.four` (rawValue 4.0) — CGFloat rawValue
- `next() -> ZoomLevel` method cycling `.one -> .two -> .four -> .one`
- This satisfies ZOOM-01 (zoom to 2x) and ZOOM-02 (cycle to 4x then back to 1x)

**ZoomState struct** (package access):
- Properties: `level: ZoomLevel = .one`, `panOffset: CGPoint = .zero`
- Computed `isZoomed: Bool` returning `level != .one`
- Computed `contentTransform: CATransform3D` that creates a scale transform then translates by panOffset. The correct composition is: `CATransform3DTranslate(CATransform3DMakeScale(s, s, 1), panOffset.x, panOffset.y, 0)` — this applies the translate in scaled-space which is what we want for capture-space pan offsets. Import QuartzCore for CATransform3D.
- `mutating func reset()` that sets level to .one and panOffset to .zero

**Coordinate mapping functions** (package-level free functions or static methods on ZoomState):

1. `windowPointToCapturePoint(_ windowPoint: NSPoint, zoomState: ZoomState, screenSize: CGSize) -> NSPoint`
   - Converts a window-local cursor point to the corresponding point in the original (unzoomed) capture
   - Formula: `captureX = (windowPoint.x / scale) - panOffset.x`, same for Y
   - At 1x with no pan, this is identity
   - Import AppKit for NSPoint

2. `capturePointToWindowPoint(_ capturePoint: NSPoint, zoomState: ZoomState) -> NSPoint`
   - Inverse: converts a capture-space point to window-space
   - Formula: `windowX = (capturePoint.x + panOffset.x) * scale`, same for Y

3. `panOffsetForZoom(cursorWindowPoint: NSPoint, currentZoom: ZoomState, newLevel: ZoomLevel, screenSize: CGSize) -> CGPoint`
   - Calculate new pan offset so cursor position remains fixed on screen when changing zoom level
   - Steps: (a) find the capture-space point under cursor using windowPointToCapturePoint, (b) solve for new pan offset: `panX = (cursorWindowPoint.x / newScale) - capturePoint.x`
   - This satisfies the "cursor stays fixed on screen during zoom" locked decision

4. `clampPanOffset(_ offset: CGPoint, zoomLevel: ZoomLevel, screenSize: CGSize) -> CGPoint`
   - Clamp pan offset so visible viewport stays within capture bounds
   - viewportW = screenSize.width / scale, viewportH = screenSize.height / scale
   - minPanX = -(screenSize.width - viewportW), maxPanX = 0. Same for Y.
   - Use max(minPan, min(0, offset)) pattern
   - At 1x, viewport = screen size so offset is clamped to (0,0) — correct identity behavior
   - This satisfies ZOOM-04 (view pans to follow cursor) and the "hard stop at screen boundary" locked decision

All types use `package` access level per project conventions (referenced within DesignRulerCore module only). Use `import QuartzCore` for CATransform3D and `import AppKit` for NSPoint/CGPoint/CGSize.

SHUX-02 (per-window zoom state) is satisfied by this being a value type struct — each OverlayWindow will own its own instance (wired in Plan 02).

Do NOT use `import Foundation` unless needed — follow existing file patterns in Utilities/.
